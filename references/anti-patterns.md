# Research Anti-Patterns (English Edition)

Use this reference to detect and correct common research failure modes.

## High-Risk Anti-Patterns
1. **Solution-first framing**
2. **Confirmation bias**
3. **Leading interview questions**
4. **Selective note-taking**
5. **Over-generalizing from one participant**
6. **Skipping synthesis discipline**
7. **No confidence labels**

## Team-Level Anti-Patterns
- Stakeholder dominance in sessions
- Role confusion between facilitator and observer
- No explicit decision owner
- Findings not tied to implementation

## Corrective Actions
- Add assumption register
- Force evidence links per claim
- Require confidence labels
- Add phase gate blockers for missing outputs
- Run lightweight peer review before decisions

## Red Flags in Reports
- Generic recommendations without evidence
- No trade-off discussion
- No owner/timeline
- No mention of limitations

## Recovery Playbook
- If data quality is weak: mark as directional
- If bias is detected: run supplemental sessions
- If scope drift occurs: reset objective and decision question

## Output Contract
- Risk register
- Corrective action list
- Updated confidence per finding
