# ux-research-agent

> Modular UX research agent built for OpenClaw, tuned for Claude-style syntheses and humanized delivery.

[![Status](https://img.shields.io/badge/status-active-success)](https://github.com/dendyadinirwana/ux-research-agent)

## TL;DR
- Phase-gated UX research workflow (Empathize → Define → Ideate → Prototype → Test).  
- Designed as an OpenClaw skill with Claude-compatible runtimes so research prompts stay evidence-backed yet fluid.  
- Includes configurable method registry, style archetypes, and quality guardrails for reproducible outputs.

## Why this exists
Research teams have too many templates and not enough discipline. `ux-research-agent` enforces rigor via configurable phase gates, method registries, and automated decision logs while letting Claude (or any OpenClaw agent) narrate the story in a voice that adapts to the stakeholder.

## Features
- **Phase gates** (config/phase-gates.yaml) that ensure each stage delivers the right artifacts before advancing.  
- **Method registry** (config/method-registry.yaml) that maps tasks to the right research techniques.  
- **Style archetypes** (`styles/*`) so responses can lean clarity-first, empathy-first, or systems-thinking while staying neutral.  
- **Quality benchmarks** (config/quality-benchmarks.yaml) enforcing labeled assumptions, explicit confidence, decision logs, and next actions.  
- **Claude + OpenClaw runtime bridge** with packaging scripts in `ports/claude/` and OpenClaw-friendly subskills ready to deploy.

## Getting started
1. **Prerequisites**: install [OpenClaw CLI](https://docs.openclaw.ai) and make sure `openclaw` is on your PATH.  
2. **Clone the repo**: `git clone https://github.com/dendyadinirwana/ux-research-agent.git`  
3. **Install dependencies** (if any) by reviewing the subskills that rely on Python/Node; there is no single monolith so follow each subskill’s README as needed.  
4. **Register the skill** with OpenClaw: drop `skills/ux-research-agent` into your workspace and run `openclaw skill install ux-research-agent` (or use your workspace automation).  
5. **Run a Claude-friendly package** when you need a zip export: `make claude-package` → `dist/ux-research-agent-claude.zip` ready for Claude deployments.

## Running inside OpenClaw + Claude explains
1. **OpenClaw orchestrates**: the skill exposes entry points (e.g., `ux-persona-builder`, `ux-decision-log`) that OpenClaw maps to user prompts and automations.  
2. **Claude-style delivery**: `ports/claude/` holds packaging docs so Claude agents can call the same skill via Claude-compatible prompts. The skill keeps evidence, references, and decision logs structured while Claude crafts humanized narratives.  
3. **Voice profiles**: configure `config/voice-tone-profiles.yaml` and `styles/` to influence whether responses feel consultative, narrative, or executive, without sacrificing guardrails.  
4. **Quality checks**: `config/quality-benchmarks.yaml` runs in every phase to catch missing assumptions, unlabeled confidence, or absent next steps before a template completes.

## Quick start example
1. Launch OpenClaw CLI.  
2. Run `openclaw skill run ux-research-agent --entry empathize --persona clarity-pragmatic`.  
3. Follow the prompts to upload notes, annotate insights, and capture the decision log.  
4. Claude/CLI will return cookies with evidence, blockers, and next actions shaped by the configured archetype.
